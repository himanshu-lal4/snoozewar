# Virtual Company — 217 AI Agent Roster

> **You are the Commander.** One human. Every role below is a virtual AI agent you invoke on demand. They have no ego, no vacation days, no skill ceiling — only a scope, a briefing requirement, and a delivery standard. Deploy them. Chain them. Run them in parallel. Review their output. Make the call.

> **The product:** A chronotype-aware behavior enforcement app (React Native/Expo, Node.js, MongoDB). Markets: US, India, UK. You orchestrate everything.

## Commander's Interface

### How to Think About Agents

Each agent is a **lens** — a way to force yourself (or your AI tool) into a specific expertise mode with specific quality standards. When you invoke an agent, you are not "pretending" — you are constraining the problem space to get a deeper, more expert output than generalist thinking produces.

**Agent Levels:**
- **Strategic** — Sets vision, defines "what" and "why" across the entire company or department. Thinks in quarters and years.
- **Tactical** — Translates strategy into plans, makes trade-offs, coordinates across functions. Thinks in weeks and months.
- **Execution** — Does the deep work. Builds, creates, designs, writes. Thinks in days and weeks.
- **Task** — Implements specific, well-scoped work items. Thinks in hours and days.
- **Research** — Gathers data, analyzes patterns, surfaces insights. Feeds other agents with raw material.

### Invoking Agents

When you sit down to work, ask: **"Which agent am I right now?"**

- **Single agent:** "I need to write paywall copy." → Invoke **Senior Copywriter Agent**.
- **Agent chain:** "I need to launch a feature." → Chain: PM Agent → UX Designer Agent → Mobile Engineer Agent → QA Agent → ASO Agent.
- **Parallel agents:** "I need to prepare for a pricing decision." → Run in parallel: Pricing Analyst Agent (competitor research) + Financial Analyst Agent (revenue modeling) + Data Analyst Agent (user behavior).

### Chaining Rules

1. **Sequential chain:** Agent A's output becomes Agent B's required input. Do not skip the chain — each agent adds a layer of quality.
2. **Parallel fan-out:** When multiple agents need the same input (e.g., a product brief), run them simultaneously and synthesize at the end.
3. **Review gate:** After every chain, the Commander reviews. No agent's output ships without your sign-off.
4. **Escalation:** If a Task agent's output doesn't meet the bar, escalate to the Execution agent for that function. If Execution fails, escalate to Tactical.

| **Building a feature** | CPO Agent (prioritize) → Senior PM Agent (spec) → Head of UX Research Agent (validate need) → Lead UX Designer Agent (wireframe) → Lead UI Designer Agent (high-fi) → Senior Mobile Engineer Agent (build) → QA Lead Agent (test) → ASO Lead Agent (store listing) |

## How the Commander Uses This

1. **Before working:** Ask "Which agent am I deploying?" Read that agent's definition. Adopt its scope, not broader, not narrower.

2. **Briefing yourself:** Check "Agent Receives" — do you have what the agent needs? If not, invoke the upstream agent first.

3. **Quality check:** After completing work, check "Quality Gate." If you can't answer yes, the output isn't ready.

4. **Delegation to AI tools:** Copy the agent definition into your AI tool's prompt. The "Agent Receives" tells you what context to provide. The "Agent Delivers" tells you what to expect back. The "Quality Gate" tells you how to evaluate the output.

5. **Stage priorities:** You cannot run 217 agents simultaneously. Deploy what matters now:
   - **Pre-launch:** CEO, CPO, CTO, Lead UX, Lead UI, Senior Mobile Engineer, Head of Copy, ASO Lead
   - **Launch:** CMO, Head of Growth, Head of Content, Social Media Director, Head of Support
   - **Scale:** CFO, Head of Data, Head of Experimentation, Head of Revenue, Head of Community

6. **Completeness sweep:** Before any launch, scan every department header. Is there an agent that would flag something? Run it.
